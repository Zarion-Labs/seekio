  # Security Policy

    ## Reporting a Vulnerability

    If you discover a security vulnerability in seekio, please **do not open a public issue**. Instead, report it privately by emailing:

    **security@zarionlabs.com**

    Include:
    - A description of the issue and potential impact
    - Steps to reproduce
    - Affected version / commit SHA if known

    ### What to expect

    - Initial acknowledgement within 3 business days.
    - A triage update within 7 business days with our assessment.
    - If confirmed: a coordinated fix and, where appropriate, a credit in the release notes.
    - If declined: a clear explanation of why.

    ## Scope

    In scope:
    - seekio itself (`seekio.js`, `public/index.html`)
    - Shipped launch scripts and platform adapters

    Out of scope:
    - Third-party dependencies (report upstream)
    - Vulnerabilities that require already-privileged local access with no remote component
    - Issues affecting only outdated forks
